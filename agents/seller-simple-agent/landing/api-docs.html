API Documentation

Integrate ComplianceShield into your workflow. Scan content for FTC, COPPA, GDPR, and platform-specific violations through REST, A2A, or MCP.

Base URL http://localhost:9000

🔑 Authentication

The REST API demo endpoints are open and require no authentication. For the A2A protocol, payment is handled via the Nevermined x402 protocol using a payment-signature header.

ℹ️
Free Demo: All /api/* REST endpoints are currently free with no API key required. Rate limits apply.
💡
A2A Paid Access: Agent-to-agent calls require a Nevermined payment token. Obtain one from nevermined.app and pass it as a payment-signature header.

⏱ Rate Limits

Demo API endpoints have generous rate limits for hackathon use.

Endpoint Rate Limit Notes
/api/scan 60 req/min Text scanning
/api/scan-video 10 req/min YouTube analysis (slower due to transcription)
/api/scan-upload 10 req/min File upload + Whisper transcription
/api/stats 120 req/min Lightweight read

⚡ REST API

Free demo endpoints for scanning text, YouTube videos, and uploaded media files. No authentication required.

POST /api/scan Scan text content ▶

Analyze text content for legal compliance issues. Detects FTC disclosure violations, COPPA concerns, GDPR issues, health/financial claims, and platform-specific policy violations.

Request Body (JSON)
ParameterTypeRequiredDescription
content string required The text content to scan for compliance issues
detail_level string optional Analysis depth. One of: quick, full, deep
Default: "quick" 
curl -X POST http://localhost:9000/api/scan \
  -H "Content-Type: application/json" \
  -d '{
    "content": "Use code SAVE20 for 20% off this amazing supplement! It cured my anxiety and depression. Perfect for kids too!",
    "detail_level": "full"
  }'
import requests

response = requests.post(
    "http://localhost:9000/api/scan",
    json={
        "content": "Use code SAVE20 for 20% off! It cured my anxiety. Perfect for kids!",
        "detail_level": "full"
    }
)

result = response.json()
print(result["report"]["risk_score"])  # e.g., 85
print(result["report"]["violations"])  # list of detected issues
const response = await fetch("http://localhost:9000/api/scan", {
  method: "POST",
  headers: { "Content-Type": "application/json" },
  body: JSON.stringify({
    content: "Use code SAVE20 for 20% off! It cured my anxiety. Perfect for kids!",
    detail_level: "full"
  })
});

const { report } = await response.json();
console.log(report.risk_score);   // e.g., 85
console.log(report.violations);   // array of issues

POST /api/scan-video Scan YouTube video ▶

Analyze a YouTube video for compliance issues. Automatically fetches the transcript and scans it for FTC, COPPA, GDPR, and platform violations. Returns video metadata alongside the compliance report.

Request Body (JSON)
ParameterTypeRequiredDescription
youtube_url string required Full YouTube URL (e.g., https://youtube.com/watch?v=abc123)
detail_level string optional Analysis depth: quick, full, or deep
Default: "quick" 
curl -X POST http://localhost:9000/api/scan-video \
  -H "Content-Type: application/json" \
  -d '{
    "youtube_url": "https://www.youtube.com/watch?v=dQw4w9WgXcQ",
    "detail_level": "full"
  }'
import requests

response = requests.post(
    "http://localhost:9000/api/scan-video",
    json={
        "youtube_url": "https://www.youtube.com/watch?v=dQw4w9WgXcQ",
        "detail_level": "full"
    }
)

data = response.json()
print(f"Video: {data['video_metadata']['title']}")
print(f"Risk:  {data['report']['risk_score']}/100")

POST /api/scan-upload Upload audio/video file ▶

Upload an audio or video file for compliance scanning. The file is transcribed using OpenAI Whisper, then the transcript is analyzed for compliance violations.

⚠️
Max file size: 25 MB. Files larger than 25 MB will be rejected with a 413 error.
Supported File Formats
MP3 MP4 WAV WebM M4A OGG FLAC
Request (Multipart Form Data)
ParameterTypeRequiredDescription
file binary required Audio or video file to upload and scan
detail_level string (query) optional Analysis depth: quick, full, or deep
Default: "quick" 
curl -X POST "http://localhost:9000/api/scan-upload?detail_level=full" \
  -F "file=@my-video.mp4"
import requests

with open("my-video.mp4", "rb") as f:
    response = requests.post(
        "http://localhost:9000/api/scan-upload",
        params={"detail_level": "full"},
        files={"file": ("my-video.mp4", f, "video/mp4")}
    )

data = response.json()
print(f"Transcribed {data['transcript_length']} characters")
print(f"Risk score: {data['report']['risk_score']}")

🤖 A2A Protocol (Agent-to-Agent)

The A2A protocol enables autonomous agent-to-agent communication with built-in payment via Nevermined x402. Ideal for integrating ComplianceShield into AI agent workflows.

💳
Paid Access: A2A endpoints require credits purchased through nevermined.app. Each skill costs a different number of credits. Pass the payment token in the payment-signature header.

Sending a Message (JSON-RPC)

Use the message/send JSON-RPC method to invoke skills. Include the payment-signature header with your Nevermined x402 access token. 

curl -X POST http://localhost:9000/ \
  -H "Content-Type: application/json" \
  -H "payment-signature: <your-nevermined-x402-token>" \
  -d '{
    "jsonrpc": "2.0",
    "method": "message/send",
    "id": "req-001",
    "params": {
      "message": {
        "role": "user",
        "parts": [
          {
            "kind": "text",
            "text": "Check this for compliance: Buy my course now! Limited spots! I made $50k in one month with this system, you can too!"
          }
        ]
      },
      "metadata": {
        "skill_id": "full_analysis"
      }
    }
  }'
import requests

NVM_TOKEN = "your-nevermined-x402-token"
AGENT_URL = "http://localhost:9000/"

response = requests.post(
    AGENT_URL,
    headers={
        "Content-Type": "application/json",
        "payment-signature": NVM_TOKEN,
    },
    json={
        "jsonrpc": "2.0",
        "method": "message/send",
        "id": "req-001",
        "params": {
            "message": {
                "role": "user",
                "parts": [{
                    "kind": "text",
                    "text": "Check this for compliance: Buy my course now!"
                }]
            },
            "metadata": {"skill_id": "full_analysis"}
        }
    }
)

result = response.json()
print(result["result"]["artifacts"])

🔌 MCP Server (Model Context Protocol)

Connect ComplianceShield as a tool in Claude Desktop, Cursor, or any MCP-compatible client. Your AI assistant gains the ability to scan content for compliance issues directly.

Setup — Claude Desktop

Add the following to your Claude Desktop configuration file at claude_desktop_config.json: 

{
  "mcpServers": {
    "compliance-shield": {
      "url": "http://localhost:3000/mcp"
    }
  }
}
📁
Config file locations:
macOS: ~/Library/Application Support/Claude/claude_desktop_config.json
Windows: %APPDATA%\Claude\claude_desktop_config.json
Linux: ~/.config/claude/claude_desktop_config.json
✅
Remote access: To connect to the hosted instance instead of localhost, use:
"url": "http://localhost:9000/mcp"
Available Tools

Once connected, the following tools become available in your AI assistant:

quick_scan
Fast compliance check for text content. Returns risk score and top violations.
full_analysis
Comprehensive scan across FTC, COPPA, GDPR and platform rules.
deep_review
Expert legal review with citations, severity ratings, and remediation steps.
scan_video
Provide a YouTube URL to scan the video transcript for compliance.
Example Usage in Claude
// In Claude Desktop, simply ask:

"Can you do a compliance check on this text:
 This supplement will cure your insomnia! Use code SLEEP20
 for 20% off. Great for kids and adults!"

// Claude will automatically use the quick_scan tool and return:
// - Risk score and level
// - Detected violations (FTC health claims, COPPA, disclosure)
// - Recommended fixes

🚨 Error Codes

Standard HTTP status codes are used. Error responses include a JSON body with details.

CodeMeaningDescription
200 OK Request succeeded
400 Bad Request Missing or invalid parameters (e.g., no content field)
402 Payment Required A2A endpoint requires a valid payment-signature
413 Payload Too Large Uploaded file exceeds 25 MB limit
415 Unsupported Media Type File format not supported (must be MP3, MP4, WAV, WebM, M4A, OGG, or FLAC)
429 Too Many Requests Rate limit exceeded. Retry after the indicated period.
500 Internal Server Error Server-side error. Check /api/health for status.
Error Response Format
{
  "status": "error",
  "error": "Missing required field: content",
  "code": 400
}
