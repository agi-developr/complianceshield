For Developers
REST API quickstart
Integrate compliance scanning into your app with a single API call. Full docs at /api-docs.
bash
# Scan text content for compliance issues curl -X POST https://api.complianceshield.ai/scan \ -H "Content-Type: application/json" \ -H "Authorization: Bearer YOUR_API_KEY" \ -d '{ "content": "This supplement is clinically proven to cure diabetes!", "content_type": "text", "jurisdictions": ["US", "EU"] }' # Response { "risk_score": 23, "issues": [ { "type": "health_claim", "severity": "critical", "regulation": "FDA 21 CFR 101.93", "suggestion": "Remove 'clinically proven to cure' -- use 'may support' instead" } ] }
python
import requests response = requests.post( "https://api.complianceshield.ai/scan", headers={"Authorization": f"Bearer {API_KEY}"}, json={ "content": "Check out this amazing weight loss product! #ad", "content_type": "text", "jurisdictions": ["US"], }, ) result = response.json() print(f"Risk Score: {result['risk_score']}/100") for issue in result["issues"]: print(f" [{issue['severity']}] {issue['type']}: {issue['suggestion']}")
javascript
const response = await fetch("https://api.complianceshield.ai/scan", { method: "POST", headers: { "Content-Type": "application/json", "Authorization": `Bearer ${API_KEY}`, }, body: JSON.stringify({ content: "Subscribe for more health tips! Link in bio.", content_type: "text", jurisdictions: ["US", "EU"], }), }); const { risk_score, issues } = await response.json(); console.log(`Risk: ${risk_score}/100 | Issues: ${issues.length}`);

Let your AI agents call ComplianceShield natively through the Agent-to-Agent protocol or as a Model Context Protocol tool.

A2A Agent Discovery A2A

ComplianceShield publishes a standard agent card for autonomous discovery. Point any A2A-compatible agent at our well-known URL.

Agent Card URL
https://complianceshield.ai/.well-known/agent.json

Claude Desktop MCP Config MCP

Add ComplianceShield as a tool in Claude Desktop. Paste this into your claude_desktop_config.json:

json
{ "mcpServers": { "complianceshield": { "command": "npx", "args": ["-y", "@complianceshield/mcp-server"], "env": { "COMPLIANCESHIELD_API_KEY": "YOUR_API_KEY" } } } }

Once configured, Claude can call scan_content, get_report, and list_regulations as native tools.

FAQ
Common questions
Quick answers to the things people ask most.
We check against FTC endorsement guidelines, FDA health claim regulations (21 CFR), GDPR data disclosure requirements, FCC rules for broadcast content, COPPA for child-directed content, and platform-specific policies for YouTube, TikTok, Instagram, and Twitter. New regulations are added continuously.
Content is processed in memory and discarded after analysis. We store only the compliance report metadata (risk scores, issue types) for your dashboard history. Full content is never retained, logged, or used for model training. Enterprise plans offer a zero-retention mode.
Our models achieve 94% accuracy on FTC disclosure detection and 91% on health claim identification based on internal benchmarks. We recommend treating reports as advisory -- ComplianceShield augments legal review but does not replace it. Critical content should always be reviewed by a qualified attorney.
